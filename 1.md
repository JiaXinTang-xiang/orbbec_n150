# Orbbec YOLO Project

基于 Orbbec Astra Mini S 深度相机 + YOLO 的目标检测与三维测距系统。

从 D435 + pyrealsense2 迁移至 Orbbec + libOrbbecSDK.so (v1.10.27)。

## 功能

- Orbbec 深度相机获取 RGB + Depth 双流
- 硬件 D2C (Depth-to-Color) 对齐
- YOLO 目标检测（支持 CUDA）
- 抗灯光干扰过滤（灰度方差）
- 像素坐标 → 3D 世界坐标转换
- PID 追踪控制（可选）
- 串口通讯（可选，MTI 19 字节协议）
- 相机内参保存

## API 说明

### OrbbecCamera

```python
from orbbec_camera import OrbbecCamera

camera = OrbbecCamera(width=640, height=480, fps=30)

# 获取对齐的 RGB + Depth 帧
color_image, depth_image, depth_data = camera.get_frames()
# color_image:  BGR numpy array (H, W, 3)
# depth_image:  uint16 numpy array (H, W), raw*pixel_scale = mm
# depth_data:   same as depth_image

# 像素坐标 -> 3D 坐标
point = camera.get_3d_point(x, y, depth_data)
# point: {'x': m, 'y': m, 'z': m, 'distance': m} or None

# 深度图可视化
colormap = camera.get_depth_colormap(depth_image, min_depth=0.0, max_depth=10.0)

# 停止
camera.stop()
```

### 深度值说明

| SDK | 原始值含义 | 比例因子 | 如何得米 |
|-----|-----------|----------|---------|
| D435 (pyrealsense2) | 米 | `depth_scale` (e.g. 0.001) | `raw * depth_scale` = 米 |
| Orbbec (libOrbbecSDK) | 无单位 | `getValueScale()` | `raw * scale / 1000` = 米 |

Orbbec 的 `getValueScale()` 返回 raw→mm 的因子。例如 depth_scale=1.0 时，raw 值即毫米。

## D435 → Orbbec 迁移对照

| 功能 | D435 (pyrealsense2) | Orbbec (libOrbbecSDK) |
|------|---------------------|-----------------------|
| 打开设备 | `pipeline.start(config)` | `pipeline_start_with_config()` |
| 获取帧 | `pipeline.wait_for_frames()` | `pipeline_wait_for_frameset()` |
| D2C 对齐 | `rs.align(rs.stream.color)` | `ALIGN_D2C_HW_MODE` |
| 内参 | `frame.profile.intrinsics` | `pipeline_get_camera_param()` |
| 深度标尺 | `sensor.get_depth_scale()` | `depth_frame_get_value_scale()` |
| 像素→3D | `rs2_deproject_pixel_to_point()` | 手动计算 `(u-cx)*Z/fx` |
| 深度距离 | `depth_frame.get_distance(x, y)` | `depth_image[y, x] * scale / 1000` |

## 测距方法

### 为什么不能开 D2C 对齐？

Astra Mini S 开启 D2C (Depth-to-Color) 后，SDK 会对深度图做扭曲映射使其对齐到彩色图。但 Astra Mini S 的 depth 和 color 摄像头基线较大，对齐后的深度图会产生大量空洞和伪值，导致目标位置的深度数据不可靠（始终为 0 或被扭曲）。

**结论：关掉 D2C，depth 和 color 各取各的，用深度内参直接算 3D 坐标。**

### 正确方法

#### 1. 关闭 D2C

```python
ob_config_set_align_mode(config, ALIGN_DISABLE, error)
# depth 和 color 各自独立输出，不做对齐
```

#### 2. 区域采样避免空洞

D2C 虽然关了，但深度图本身也可能有局部空洞（物体边缘、反光面等）。单点读取容易命中 0 值。

```python
# 在目标中心周围 11x11 区域采样，取非零值的中位数
r = 5
region = depth_frame[y-r:y+r+1, x-r:x+r+1]
valid = region[region > 0]
depth_mm = np.median(valid)  # mm
```

#### 3. 用深度内参做 3D 转换

因为没开 D2C，深度像素坐标用的是**深度摄像头**的内参，不是 RGB 内参：

```
X = (u - cx_depth) * Z / fx_depth
Y = (v - cy_depth) * Z / fy_depth
Z = depth_raw * depth_scale / 1000.0   (mm -> m)
distance = sqrt(X^2 + Y^2 + Z^2)
```

代码实现 (`orbbec_camera.py`):

```python
def get_3d_point(self, x, y, depth_frame_data, sample_radius=5):
    # 区域采样取中位数
    h, w = depth_frame_data.shape
    y1, y2 = max(0, y - r), min(h, y + r + 1)
    x1, x2 = max(0, x - r), min(w, x + r + 1)
    region = depth_frame_data[y1:y2, x1:x2]
    valid = region[region > 0]
    if len(valid) == 0:
        return None

    depth_raw = float(np.median(valid))
    z_mm = depth_raw * self.depth_scale  # -> mm
    z_m = z_mm / 1000.0                  # -> m
    if z_m <= 0:
        return None

    # 使用深度内参 (非 RGB 内参)
    intr = self.depth_intrinsics
    x_m = (x - intr.ppx) * z_m / intr.fx
    y_m = (y - intr.ppy) * z_m / intr.fy

    dist = np.sqrt(x_m**2 + y_m**2 + z_m**2)
    return {'x': x_m, 'y': y_m, 'z': z_m, 'distance': dist}
```

#### 4. 深度标尺

Orbbec SDK 的 `getValueScale()` 返回 raw → mm 的因子：

| 设备 | depth_scale | raw=1000 含义 |
|------|-------------|--------------|
| Astra Mini S | 1.0 | 1000mm = 1m |
| 其他设备 | 0.1 | 100mm = 0.1m |

### 为什么不直接用 RGB 内参？

D435 开了 `rs.align` 后，深度图被重映射到 RGB 坐标系，所以用 RGB 内参是对的。

Orbbec 关掉 D2C 后，深度图保持**深度摄像头自身的坐标系**，必须用深度内参。Color 和 depth 两个摄像头之间有物理基线偏移，但分辨率相同 (640x480)，所以像素坐标可以直接对应，误差可接受。
